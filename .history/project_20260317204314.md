# Project Overview: `platform`

## What this folder is
This folder contains the **Deep Track Platform web application**, built with **Next.js 15 (App Router)** and **TypeScript**.

It is a single full-stack Next.js project:
- **Frontend UI** is implemented with React components in `app/`, `components/`, and `modules/`
- **Backend endpoints** (server-side handlers) live under `app/api/`
- **Shared logic/helpers** are in `lib/`, `utils/`, `hooks/`, and `actions/`

## Main tech stack
- `next` + `react` + `typescript`
- `tailwindcss` for styling
- `@clerk/nextjs` for authentication
- `@tanstack/react-table`, `recharts`, `react-hook-form`, `zod` and other UI/data libs
- `eslint` + `biome` for code quality/formatting support

## Important folders
- `app/` - App Router pages, layouts, route groups, and API routes
  - `app/(clerk-auth)/` - auth-related route group
  - `app/(onboard)/` - onboarding flow route group
  - `app/(platform)/` - main platform route group
  - `app/api/` - server/API handlers
- `components/` - reusable UI components (`components/ui` for primitive UI pieces)
- `modules/` - domain feature modules (e.g., `kyc`, `organization`)
- `actions/` - server/client actions and business operations
- `lib/`, `utils/`, `hooks/` - shared utilities, libraries, and custom hooks
- `public/` - static assets
- `fonts/` - local/project fonts

## Key config files
- `package.json` - scripts and dependencies
- `tsconfig.json` - TypeScript config
- `next.config.ts` - Next.js config
- `tailwind.config.ts` and `postcss.config.mjs` - styling pipeline
- `eslint.config.mjs` and `biome.json` - lint/format config
- `bun.lock` - Bun lockfile (project appears Bun-oriented)

## Common scripts
From `package.json`:
- `dev` -> `next dev --turbopack`
- `build` -> `next build`
- `start` -> `next start`
- `lint` -> `next lint`

## Notes
- This repository currently looks like a **single app** (no separate backend service folder detected).
- If you need both “backend + frontend” in local development, they are typically both served by the Next.js app (`app/` + `app/api/`) via the same dev process.
